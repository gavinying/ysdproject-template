# AI Instructions

## General Guidelines

- **PLANNING MODE BY DEFAULT**: Always stay in planning mode. Do NOT execute any plan or make code changes until the user explicitly instructs you to proceed (e.g., "go ahead", "execute", "proceed", "do it").
- **Always ask user for approval before executing plans**
- Create `implementation_plan.md` for significant changes
- When in doubt, ask for clarification
- **Documentation**: Use `context7` MCP server to query latest documentation for libraries (especially Better Auth, Prisma, Drizzle, etc.).
- use yaml/toml instead of json for configuration files
- use pnpm instead of npm for package management
- **Git Workflow**: After completing a feature or fix, describe and commit changes to the local git repository. Do NOT push to remote - let the user handle that.
